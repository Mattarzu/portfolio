SERVERLESS · CONTACTO · AUTOMATIZACIÓN

M M LAB Contact Worker

Backend serverless para conectar formularios públicos con notificaciones reales, manteniendo secretos fuera del repositorio y separando frontend estático de lógica sensible.

Cloudflare Worker Telegram Bot API CORS JavaScript Serverless

Problema

Un portfolio estático necesita recibir mensajes reales sin exponer tokens ni montar un backend tradicional.

  • Formulario público conectado a un endpoint seguro.
    • \n
  • Token del bot fuera de Git mediante secrets.
    • \n
  • CORS limitado a origen productivo.
    • \n

Arquitectura

La solución separa GitHub Pages como frontend y Cloudflare Worker como backend liviano de contacto.

  • Frontend estático con configuración pública del endpoint.
    • \n
  • Worker con validación y envío a Telegram.
    • \n
  • Documentación operativa para mantenimiento y despliegue.
    • \n
Tipo Backend serverless productivo.
Uso Contacto real desde Panda / formulario web.
Valor Permite operar una web estática con integración real y bajo costo.
